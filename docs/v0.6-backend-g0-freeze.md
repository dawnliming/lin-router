# v0.6 后端 G0：执行平面设计冻结

- 状态：已冻结，进入 I1
- 范围：后端内部无行为重构；`frontend/` 不属于本变更集
- 基线：`ea534bc`（v0.5.6 发布元数据）

## 1. 依赖方向

```text
HTTP Transport / ArkProxyRouter compatibility facade
  -> NonStreamExecutionService / StreamExecutionService
  -> contracts ports + runtime DTO / errors
  -> ConfigStore | SettingsStore | ObservabilityService | WafLockState | UpstreamClient | UpstreamAdapter
```

`runtime/`、`contracts/` 与 `upstream/` 禁止反向导入 `app.py`、`RouterHandler`。执行服务接收显式依赖；HTTP transport 只解析/序列化请求与消费流迭代器。

## 2. 最小端口与唯一 owner

| Port | 方法级契约 | 唯一 owner |
|---|---|---|
| CandidateQueryPort | `iter_upstream_candidates`、`iter_aggregate_candidates`、`resolve_aggregate` | ConfigStore 查询适配器 |
| HealthStatePort | `refresh_expired_cooldowns`、`mark_success`、`mark_unusable`、`set_cooldown`、`set_aggregate_member_cooldown` | ConfigStore |
| UpstreamRequestPort | `request`、`close` / 装配层刷新 | UpstreamClient；开关 owner 为 SettingsStore |
| ConcurrencyPort | `lock_for`、`active_count`、`mark_stream_active` | WafLockState（进程内易失） |
| RequestObservabilityPort | `start_live_request`、`update_live_request`、`finish_live_request`、`add_log`、`patch_stream_lifecycle` | ObservabilityService；DebugCapture 是协作者 |
| ProtocolAdapterPort | `resolve_endpoint`、`build_request`、`fetch_models` | UpstreamAdapter |
| ExecutionPolicyPort | 分类、auto/aggregate 判断、body/usage/脱敏详情 | 无状态策略，不持有 Store 或 HTTP Client |

短生命周期 `response`、首包、iterator、锁持有权由执行服务负责；首包成功后流式服务负责关闭 response、写最终生命周期、释放锁。Handler 仅在 `finally` 调用 `finalize_stream_if_needed` 兜底。

## 3. 兼容 facade 与装配

核心 DTO/error：`UpstreamCandidate`、`RouteContext`、`AllModelsFailedError`、`StreamIdleTimeoutError` 位于 `linrouter_core/contracts/runtime_types.py`。

`ArkProxyRouter` 保留公开 `call`、`stream`、`finalize_stream_if_needed` 与既有诊断/模型/配置辅助入口：

- `call` 只委托 `NonStreamExecutionService.execute`；
- `stream` 只委托 `StreamExecutionService.execute`；
- 不保留候选循环、fallback、真实 HTTP 请求、流读取或健康状态写入；
- `app_runtime.create_application_server`（或其 composition factory）装配 Store、Settings、观测、并发、HTTP client/adapter、服务、facade、HTTP server。

## 4. 无行为硬边界

不改 `/api/*`、`/v1/*`、`/chat/*`、`/health` 契约；不改配置 JSON/迁移、候选排序、聚合策略、cooldown、WAF 锁、普通 fallback、流式首包与生命周期、稳定 machine `type/code` 与中文可见错误、`static/` 及 `frontend/`。

## 5. I1 进入/验收条件

- 新执行模块无 `import app`、无 `RouterHandler`、无 `router: Any`；
- 新执行 DTO/error 不从 `app.py` 取得；
- `ConfigStore`、`ObservabilityService`、`WafLockState`、`UpstreamClient`、`SettingsStore` 状态没有第二副本；
- 保留 facade 的参数、返回值、流消费/cleanup 语义；
- 全量、专项、真实 smoke、源码边界审计与独立 Codex 复核均在 G1 完成；
- `frontend/` 始终不进入后端 diff / commit。
